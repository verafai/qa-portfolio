Hi, welcome to my Manual QA portfolio.

Project 01

The software under test is the MODA OPERANDI website, which is an online luxury fashion retailer that allows customers to preorder looks directly from designers, immediately after their runway show.

This project focuses on showcasing my abilities related to creating a test plan, writing user stories and the analysis, design and execution of test cases. Therefore, only a few product features are considered.

Testing Scope

In scope

  • All the features covered in the user stories and everything specified in the acceptance criteria

    • Out of scope

  • Visual bugs, like overlapping text, layout misalignment, etc

  • Security or performance testing

    • Bug Management

    • User Stories

    ID Title Description Acceptance Criteria Priority
    1 Register | Create a user account with an email As a user, I want to create a customer account, to have a personalized service Scenario #1: User successfully creates an account

    Given the user is in the Register section
    When the user completes the registration form with valid data and clicks the ‘Register Now’ button
    Then a confirmation email is sent to the user

    Scenario #2: User cannot create an account

    Given the user is in the Register section
    When the user completes the form with invalid data
    Then an error message is displayed per field
    And nothing happens when the user clicks the ‘Register Now’ button

    Medium
    2 Sign in | Sign in and log out with a user account As a client, I want to sign in and log out of the Moda Operandi website, to have a seamless shopping experience Scenario #1: User successfully signs in

    Given the user is in the Sign in section
    And already has a verified customer account
    When the user completes the sign in form with valid data and clicks the ‘Sign in’ button
    Then the user is redirected to the homepage

    Scenario #2: User cannot sign in

    Given the user is in the Sign in section
    And already has a verified customer account
    When the user completes the sign in form with invalid data and clicks the ‘Sign in’ button
    Then an error message is displayed

    Scenario #3: User successfully logs out

    Given the user has signed in with its customer account
    When the user clicks the ‘log out’ button in the drop down menu of his profile
    Then the session successfully ends

    High
    3 PDP| Adding products to the Shopping Bag As a customer, I want to add items to the Shopping Bag, to keep a track of the items I Intend to buy Scenario #1: Customer adds an item without size option to the Shopping Bag from the PDP

    Given the customer is in the PDP of an item without size option
    When the customer clicks the ‘Add to bag’ button
    Then the item is successfully added to the Shopping Bag
    And a pop up showcasing the product appears with the text ‘Item Added’

    Scenario #2: Customer adds an item with size option to the Shopping Bag from the PDP

    Given the customer is in the PDP of an item with size option
    When the customer selects the size
    And clicks the ‘Add to bag’ button
    Then the item is successfully added to the Shopping Bag
    And a pop up showcasing the product appears with the text ‘Item Added’

    High
    4 Shopping Bag | Change the quantity of an item As a customer, I want to change the quantity of an item in the Shopping Bag, to have a seamless shopping experience Scenario #1: Customer changes the quantity of an item displayed in the Shopping Bag

    Given the customer has added an item to the Shopping Bag
    When the customer selects a quantity from the dropdown list
    Then the units of the item are modified accordingly
    And the price in the Shopping Bag section and the order subtotal of the Payment Summary section are correctly updated and match

    High
    5 Filtering | Filter the products by type and subtype As a user, I want to filter the items by type and subtype, to easily visualize them based on specified criteria Scenario #1: User selects a specific type from the navigation bar and a subtype from the dropdown menus and applies the filter

    Given the user is in the Moda Operandi homepage
    When the user selects a type (e.g. "Dresses") from the navigation bar and a subtype (e.g. “Knee-Length”) from the dropdown menu and applies the filter
    Then only items belonging to the selected type and subtype are displayed on the page
    And the URL updates to reflect the selected filter for bookmarking or sharing

    Medium
    6 Sorting | Sort the search results As a user, I want to sort the items according to 4 categories (Featured, What’s new, Price High to Low and Price Low to High) to easily visualize them Scenario #1: User selects a specific category from the “Sort by” dropdown list

    Given the user is in a Product Listing Page (PLP)
    When the user selects a category from the “Sort by” dropdown list
    Then only items belonging to the selected category are displayed on the page

    Medium
